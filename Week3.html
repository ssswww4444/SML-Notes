Lecture 5: Regularisation

Irrelevant / Multicolinear features

  • Co-linearity between features
  • Features not linearly independent
    • E.g. If x1x_1x1​ and x2x_2x2​ are the same ⇒\Rightarrow⇒ perfectly correlated
  • For linear model, feature XjX_jXj​ is irrelevant if
    • XjX_jXj​ is a linear combination of other columns

    X.j=Σl≠jαlX.l X_{.j} = \Sigma_{l \neq j} \alpha_{l}X_{.l} X.j​=Σl​=j​αl​X.l​

    for some scalars αl\alpha_lαl​
    • Equivalently: Some eigenvalue of X′XX'XX′X is zero
  • Problems
    1. The solution is not unique
      • Infinite number of solutions
    2. Lack of interpretability
      • cannot interpret the weights
    3. Optimising to learn parameter is ill-posed problem

Ill-posed problems

  • Well-posed problem
    1. a solution exists
    2. the solution is unique
    3. the solution's behavior changes continuously with the initial condition
  • If ill-posed, there is no closed form solution
    • Closed form solution w^=(X′X)−1X′y\hat{w} = (X'X)^{-1}X'yw^=(X′X)−1X′y
    • But if irrelevant, X′XX'XX′X has no inverse (singular)
    • (Even near-irrelevance / colinearity can be problematic)

Re-conditioning the problem (Ridge Regression)

  • Make it a well-posed solution and also prevent fitting noise / overfitting
  • Original problem: minimise squared error

∣∣y−Xw∣∣22 || y - Xw ||_2^2 ∣∣y−Xw∣∣22​

  • Regularised problem (L2, Ridge regression): minimise

∣∣y−Xw∣∣22+λ∣∣w∣∣22 for λ>0 || y - Xw ||_2^2 + \lambda||w||_2^2 \text{ for } \lambda > 0 ∣∣y−Xw∣∣22​+λ∣∣w∣∣22​ for λ>0

  • Turns the ridge into a peak (⇒\Rightarrow⇒ unique solution)
  • Adds λ\lambdaλ to eigenvalues of X′XX'XX′X: makes invertible

Regulariser as a prior (Bayesian intepretation of ridge regression)

  • Let prior distribution be:

W∼N(0,1/λ) W \sim N(0, 1/\lambda) W∼N(0,1/λ)

  • Higher λ\lambdaλ, more confidence at prior, therefore ignore data more
  • Computing posterior and take MAP

log(posterior)=log(likelihood)+log(prior)−log(marg) \text{log}(posterior) = \text{log}(likelihood) + \text{log}(prior) - \text{log}(marg) log(posterior)=log(likelihood)+log(prior)−log(marg)

  • can just ignore log(marg)\text{log}(marg)log(marg), since this term doesn't affect optimisation
  • Arrive at the problem of minimising:

∣∣y−Xw∣∣22+λ∣∣w∣∣22 || y - Xw ||_2^2 + \lambda||w||_2^2 ∣∣y−Xw∣∣22​+λ∣∣w∣∣22​

  • Become equivalent problem: Ridge Regression

Regularisation in non-linear models

  • There is trade-off between overfitting and underfitting
  • Right model class Θ\ThetaΘ will sacrifice some traininig error, for test error

    2. Regularisation

Explicit model selection

  • Using hold-out or CV to select the model
  1. Split data into DtrainD_{train}Dtrain​ and DvalidateD_{validate}Dvalidate​ sets
  2. For each degree d (# of parameters), we have model fdf_dfd​
    1. Train fdf_dfd​ on DtrainD_{train}Dtrain​
    2. Test (evaluate) fdf_dfd​ on DvalidateD_{validate}Dvalidate​
  3. Pick degree d^\hat{d}d^ that gives the best test score
  4. Re-train model fd^f_{\hat{d}}fd^​ using all data (return this final model)

Lecture 6: Perceptron

Artificial Neural Network (RNN, CNN, MLP)

  • A network of processing elements
    • Each element converts inputs to output
    • Output: a (activation) function of a weighted sum of inputs (linear combination)
  • To use ANN, we need
    • To design network topology
    • Adjust weights to given data
  • Training an ANN ⇒\Rightarrow⇒ adjusting weights for training data given a pre-defined network topology

Perceptron

  • Perceptron is a binary classifier
    • s=∑i=0mxiwis = \sum_{i = 0}^m x_i w_is=∑i=0m​xi​wi​
      • Predict class A if s≥0s \geq 0s≥0
      • Predict class B if s<0s < 0s<0
  • Loss function
    • Usually don't use 0-1 loss for training, since cannot calculate the gradient
    • Shouldn't give penalty for correctly classified examples
      • Penalty (loss) = sss
    • Can be re-written as L(s,y)=max(0,−sy)L(s,y) = max(0,-sy)L(s,y)=max(0,−sy)

Stochastic gradient descent (SGD)

  • Stochastic = Random: shuffling training examples
  • Randomly shuffle / split all training examples in B batches
  • Choose initial θ(1)\theta^{(1)}θ(1)
  • For iii from 1 to TTT (epochs)
    • For jjj from 1 to BBB (batches)
      • Do gradient descent update using data from batch jjj
  • Advantage: computational feasibility for large datasets

Perceptron training algorithm (SDG with batch size 1)

  • Choose initial guess w(0)w^{(0)}w(0), k=0k=0k=0
  • For iii from 1 to TTT (epochs)
    • For jjj from 1 to NNN (training examples)
      • Consider examples {xj,yj}\{\mathbf{x}_j, y_j \}{xj​,yj​}
      • Update: w(k++)=w(k)−η∇L(w(k))w^{(k++)} = w^{(k)} - \eta \nabla L(w^{(k)})w(k++)=w(k)−η∇L(w(k))
  • Training rule (gradient descent)
    • Correct: We have ∂L∂wi=0\frac{\partial L}{\partial w_i} = 0∂wi​∂L​=0 when sy>0sy > 0sy>0
    • Misclassified: We have ∂L∂wi=−xi\frac{\partial L}{\partial w_i} = -x_i∂wi​∂L​=−xi​ when y = 1 and s<0s < 0s<0
    • Misclassified: We have ∂L∂wi=xi\frac{\partial L}{\partial w_i} = x_i∂wi​∂L​=xi​ when y = -1 and s>0s > 0s>0
    • For sy = 0, we can do either of these (doesn't matter)
  • When classified correctly, weights are unchanged
  • When misclassified, update: w(k+1)+=−η(±x)\mathbf{w}^{(k+1)} += -\eta(\pm \mathbf{x})w(k+1)+=−η(±x)
    • ±x\pm x±x is gradient
  • Convergence theorem: if the trianing data is linearly separable, the algorithm is guaranteed to converge to a solution. This is, there exist a finite KKK s.t. L(wK)=0L(\mathbf{w}^K) = 0L(wK)=0
  • Pros and cons:
    • Pros: if data is linearly separable, the perceptron trianing algorithm will converge to a correct solution
    • Cons:
      • There are infinitely many possible solutions
      • If the data is not linearly separable, the training will fail completely rather than give some approx. solution
