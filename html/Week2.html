Lecture 3: Linear Regression & Optimisation

Linear Regression

  • Assume a probabilistic model
    • y=Xβ+ϵy = X\beta + \epsilony=Xβ+ϵ
  • Assume Gaussian noise (independent of X):
    • ϵ∼N(0,σ2)\epsilon \sim N(0, \sigma^2)ϵ∼N(0,σ2)
  • Discriminative model
    • p(y∣x)=12πσ2exp(−(y−xβ)22σ2)p(y|\mathbf{x}) = \frac{1}{\sqrt{2\pi\sigma^2}}\text{exp}(-\frac{(y-\mathbf{x}\beta)^2}{2\sigma^2})p(y∣x)=2πσ2​1​exp(−2σ2(y−xβ)2​)
  • Unknown param: β\betaβ (and σ2\sigma^2σ2)
  • MLE: choose param values that maximise the probability of observed data (likelihood)
    • "Log trick": instead of maximising likelihood, we can maximise log-likelihood (since log is strictly monotonic increasing)
  • Under this model ("normal" linear regression):
    • MLE is equivalent to minimising SSE (or RSS)

Optimization

  • Training = Fitting = Parameter estimation
  • Typical formulation (minimise loss = objective)
    • θ^∈arg min⁡θ∈ΘL(data,θ)\hat{\theta} \in \argmin_{\theta \in \Theta} L(data, \theta)θ^∈θ∈Θargmin​L(data,θ)
  • Analytic (aka. closed form) solution
    • 1st order derivatives for optimality:
      • ∂L∂θ1=...=∂L∂θp=0\frac{\partial L}{\partial \theta_1} = ... = \frac{\partial L}{\partial \theta_p} = 0∂θ1​∂L​=...=∂θp​∂L​=0
    • (Need to check 2nd derivative)
  • Approximate iterative solution (e.g. IRWLS)
    • Initialisation: choose starting guess θ(1)\mathbf{\theta}^{(1)}θ(1), set i=1i=1i=1
    • Update: θ(i+1)←SomeRule[θ(i)]\theta^{(i+1)} \leftarrow SomeRule[\theta^{(i)}]θ(i+1)←SomeRule[θ(i)], set i←i+1i \leftarrow i+1i←i+1
    • Termination: decide whether to stop
    • Go to step 2
    • Stop: return θ^≈θ(i)\hat{\theta} \approx \theta^{(i)}θ^≈θ(i)

  • 一次 update 一个 θi\theta_iθi​

    1. Choose θ(1)\theta^{(1)}θ(1) and some TTT

Gradient descent

  • Gradient denoted as ∇L=[∂L∂θ1,...,∂L∂θp]T\nabla L = [\frac{\partial L}{\partial \theta_1}, ..., \frac{\partial L}{\partial \theta_p}]^{T}∇L=[∂θ1​∂L​,...,∂θp​∂L​]T

Convex objective functions

  • "Bowl" shaped functions
  • Every local min is global min
  • Informal definition: line segment between any two points on graph of function lies above or on the graph
  • Gradient descent on (strictly) convex function guaranteed to find a (unique) global minimum

Lecture 4: Logistic Regression & Basis Expansion

Logistic Regression

  • Why not linear regression for classification?
    • Predict "Yes" if s≥0.5s \geq 0.5s≥0.5
    • Predict "No" if s<0.5s < 0.5s<0.5
    • (s=xβ^s = x\hat{\beta}s=xβ^​, estimated probability for class 1 given a data point)
    • Reason:
      • Can be susceptible (易受影响) to outliers
      • least-squares criterion looks unnatural in this setting
  • Problem: the probability needs to be between 0 and 1
  • Logistic function
    • f(s)=11+exp(−s)f(s) = \frac{1}{1+\text{exp}(-s)}f(s)=1+exp(−s)1​
  • Logistic regression model:
    • P(Y=1∣x)=11+exp(−xTβ)P(Y=1|x) = \frac{1}{1+\text{exp}(-x^T\beta)}P(Y=1∣x)=1+exp(−xTβ)1​
    • In GLM:
      • Mean: μ=P(Y=1∣x)\mu = P(Y=1 | x)μ=P(Y=1∣x)
      • Link function: g(μ)=logP(Y=1∣x)P(Y=0∣x)=η=xTβg(\mu) = log \frac{P(Y=1 | x)}{P(Y=0 | x)} = \eta = x^T\betag(μ)=logP(Y=0∣x)P(Y=1∣x)​=η=xTβ (log odds)
    • (Can use Cross-Validation to choose the threshold, usually just use 0.5)
  • Logistic regression is a linear classifier
    • Logistic regression model:
      • P(Y=1∣x)=11+exp(−xTβ)P(Y = 1 | x) = \frac{1}{1 + \text{exp}(-x^T\beta)}P(Y=1∣x)=1+exp(−xTβ)1​
    • Classification rule:
      • Class "1"
        • If P(Y=1∣x)=1exp⁡(−xTβ)>12P(Y=1 | x) = \frac{1}{\exp(-x^{T}\beta)} > \frac{1}{2}P(Y=1∣x)=exp(−xTβ)1​>21​
        • Else class "0"
      • Decision boundary (line):
        • P(Y=1∣x)=11+exp(−xTβ)=12P(Y = 1 | x) = \frac{1}{1 + \text{exp}(-x^T\beta)} = \frac{1}{2}P(Y=1∣x)=1+exp(−xTβ)1​=21​
        • Equivalently, P(Y=0∣x)=P(Y=1∣x)P(Y = 0 | x) = P(Y = 1 | x)P(Y=0∣x)=P(Y=1∣x)
      • (In higher dimensional problems, the decision boundary is a plane or hyperplane, vector β\betaβ is perpendicular to the decision boundary)

Linear v.s. logistic probabilistic models

  • Linear regression
    • Assume ϵ∼N(0,σ2)\epsilon \sim N(0, \sigma^2)ϵ∼N(0,σ2)
    • Therefore assume y∼N(Xβ,σ2)y \sim N(X\beta, \sigma^2)y∼N(Xβ,σ2)
  • Logistic regression
    • Assume y∼Bernoulli(p=logistic(xTβ))y \sim Bernoulli(p = logistic(x^{T}\beta))y∼Bernoulli(p=logistic(xTβ))

Logistic MLE

  • Doesn't have closed form solution (cannot solve ∂L∂β=0\frac{\partial L}{\partial \beta} = 0∂β∂L​=0 directly)
  • Therefore use iterative optimisation
    • E.g. Newton-Raphson, IRWLS, or gradient descent
  • Good news: it's a convex problem (if no irrelevant features) ⇒\Rightarrow⇒ guaranteed to get global minimum

Information divergence (extra)

  • To compare models with different num of params in an all-subsets search
  • May use information theoretic criterion which estimates information divergence between true model and a given candidate model (working model)
  • Best model: smallest criterion value
  • E.g. Kullback-Leibler Information
    • KL(f1,f2)=Ef1[logf2f1]=∫xlogf2f1f(x)dxKL(f_1,f_2) = E_{f_1}[log \frac{f_2}{f_1}] = \int_{x} log \frac{f_2}{f_1}f(x) dxKL(f1​,f2​)=Ef1​​[logf1​f2​​]=∫x​logf1​f2​​f(x)dx
    • Problem: don't know the true model ⇒\Rightarrow⇒ cannot compute KLKLKL
    • E.g. two binomial distribution: one for working model, one for true model
      • Choose the model minimise KLKLKL

Basis Expansion (Data transformation)

  • Extend the utility of models via data transformation
  • For linear regression:
    • Transformation data ⇒\Rightarrow⇒ make data more linear!
    • Map data onto another feature space s.t. data is linear in that space
      • x\mathbf{x}x: the original set of features
      • ϕ\phiϕ: Rm→Rk\R^m \rightarrow \R^kRm→Rk denotes transformation
      • ϕ(x)\phi(\mathbf{x})ϕ(x): new feature set
  • Polynomial regression:
    • New features:
      • ϕ1(x)=x\phi_1(x) = xϕ1​(x)=x
      • ϕ2(x)=x2\phi_2(x) = x^2ϕ2​(x)=x2
    • Quadratic regression (linear in new feature set):
      • y=w0+w1ϕ1(x)+w2ϕ2(x)=w0+w1x+w2x2y = w_0 + w_1 \phi_1(x) + w_2 \phi_2(x) = w_0 + w_1 x + w_2 x^2y=w0​+w1​ϕ1​(x)+w2​ϕ2​(x)=w0​+w1​x+w2​x2
  • Can be applied for both regression and classification
  • There are many possible choices of ϕ\phiϕ
  • Binary classification:
    • If dataset not linearly separable (non-linear problem)
    • Define transformation as:
      • ϕi(x)=∣∣x−zi∣∣\phi_i(x) = ||x - z_i||ϕi​(x)=∣∣x−zi​∣∣ (euclidean distance)
        • where ziz_izi​ is some pre-defined constants
        • Distances to each ziz_izi​ as new features

Radial basis functions (RBFs)

  • Motivated from approximiation theory
  • Sums of RBFs are used to approximate given functions
  • Radial basis functions:
    • ϕ(x)=ψ(∣∣x−z∣∣)\phi(x) = \psi(||x - z||)ϕ(x)=ψ(∣∣x−z∣∣), where zzz is a constant
  • Examples:
    • ϕ(x)=∣∣x−z∣∣\phi(x) = ||x - z||ϕ(x)=∣∣x−z∣∣
    • ϕ(x)=exp(−1σ∣∣x−z∣∣2)\phi(x) = \text{exp}(-\frac{1}{\sigma} ||x-z||^2)ϕ(x)=exp(−σ1​∣∣x−z∣∣2)

Challenges of basis expansion (data transformation)

  • One limitation: the transformation needs to be defined beforehand
    • Need to choose the size of new feature set
    • If using RBFs, need to choose ziz_izi​
  • Choosing ziz_izi​:
    1. Uniformly spaced points (grids)
    2. Cluster training data and use cluster centroids
    3. Use training data zi≡xiz_i \equiv x_izi​≡xi​ (some xix_ixi​)
    • E.g. ϕi(x)=ψ(∣∣x−xi∣∣)\phi_i(x) = \psi(||x - x_i||)ϕi​(x)=ψ(∣∣x−xi​∣∣)
    • For large datasets, this results in a large number of features

Taking the idea of basis expansion to the next level

  • One idea: to learn the transformation ϕ\phiϕ from data
    • E.g. Artificial Neural Networks
  • Another extension: use kernel trick
  • In sparse kernel machines, training dependes ony on a few data points
    • E.g. SVM
